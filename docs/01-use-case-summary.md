# Use Case Summary

## System Name
Internal HR Resume-Screening AI Assistant

## Purpose
This system is designed to support internal HR teams by summarizing resumes, extracting relevant skills and experience, and generating an initial relevance score based on job requirements.

## Intended Users
- HR recruiters
- Talent acquisition specialists
- Hiring managers at the review stage

## Inputs
- Candidate resumes
- Job descriptions
- Role-specific screening criteria

## Outputs
- Resume summaries
- Extracted skills and qualifications
- Initial relevance score or ranking
- Suggested strengths and gaps relative to the role

## Decision Type
Decision-support only.

The system is not intended to make final hiring, rejection, or shortlisting decisions without human review.

## Human Involvement
A human reviewer must assess all outputs before any hiring decision, shortlist, or rejection action is taken.

## Key Constraints
- No fully automated rejection decisions
- No use of sensitive personal attributes in screening logic
- Access limited to authorized personnel
- Outputs must be reviewed before use in decision-making
- The system must be used only for the defined HR screening support purpose

## Governance Relevance
This use case involves meaningful risk because it affects employment-related decision processes and processes personal data. As a result, governance controls, oversight mechanisms, and documentation requirements are necessary before deployment.
